HVE Primitives

A reference architecture for Hypervelocity Engineering — the practice of small, expert teams leveraging AI to ship at the pace of innovation without sacrificing quality, security, or scalability.

Hypervelocity Engineering (HVE) is a deliberate approach to embedding AI into the core of engineering work. Rather than treating AI as a bolt-on productivity hack, HVE teams build scalable, repeatable primitives — reusable agents, prompts, and skills — that compound over time. The result: multi-disciplinary teams with domain expertise deliver solutions faster while maintaining engineering rigor. This separates HVE from "vibe coding." It's not about moving fast and breaking things — it's about moving fast because nothing breaks.

💡 What are Primitives?

GitHub Copilot can be customized through primitives — files you place inside .github/ to shape how Copilot understands, generates, and automates in your repository. They are not ordered or sequential — Copilot Agent Mode acts as the macro orchestrator, composing any combination of primitives as the task requires.

Copilot Agent Mode is the macro orchestrator. It loads instructions automatically, invokes prompts, delegates to custom agents, reads skills, and runs hooks as guardrails — composing them in any order. Agentic workflows extend this to CI/CD via GitHub Actions.

Instructions 📋

Instructions ✓ used

Coding standards and conventions applied automatically by file pattern. The foundation every repo should have.

Location: .github/copilot-instructions.md (global) or .github/instructions/*.instructions.md (scoped)
Scope: Global instructions apply to all interactions; scoped instructions use applyTo globs to target specific file patterns.

In this repo:
  • copilot-instructions.md — Defines the tech stack (HTML/CSS/JS + Flask), and README requirements including Mermaid architecture diagrams.
Agent 🤖

Agents ✓ used

Custom autonomous personas with dedicated instructions, tool access, and personality. Agents execute multi-step tasks on the user's behalf.

Location: .github/agents/
File pattern: *.agent.md

In this repo:
  • prototype-builder — Rapid Prototype Engineer that turns specs/BRDs into working Flask prototypes with synthetic data and polished UI.

Key frontmatter: description, tools, argument-hint
Prompt 💬

Prompts ✓ used

Reusable prompt templates that guide Copilot through structured tasks — like filling out a form with AI.

Location: .github/prompts/
File pattern: *.prompt.md

In this repo:
  • customer-input-to-spec — Parses customer emails, meeting transcripts, or BRDs into structured, dev-friendly specs with Mermaid diagrams.

Key frontmatter: description, argument-hint, agent
Skill 🧠

Skills ✓ used

Deep domain knowledge packaged as procedural guides. Skills teach Copilot how to do something correctly — available to both Copilot Agent Mode and custom agents.

Location: .github/skills/<name>/
File pattern: SKILL.md inside a named folder
Loaded by: Copilot Agent Mode (generic) and custom agents — skills are not exclusive to any single agent.

In this repo:
  • ms-agent-framework — Procedure for scaffolding AI agents using Microsoft Agent Framework + AzureOpenAIResponsesClient.
  • deploy-to-aca — Scripts and IaC to deploy applications to Azure Container Apps with secure defaults.

Key frontmatter: name, description, argument-hint
Hook 🪝

Hooks not used

Automated actions triggered during Copilot agent sessions — enforce policy, run linters, or block unsafe operations as guardrails.

Location: .github/hooks/
File pattern: *.json

Examples from the community:
  • Block commits with hardcoded secrets
  • Enforce code style rules before generation
  • Auto-run tests after file changes

Reference: awesome-copilot/hooks
Workflow ⚡

Agentic Workflows not used

AI-powered GitHub Actions automations defined in markdown with natural language instructions. The CI/CD layer of Copilot customization.

Location: .github/workflows/
File pattern: *.md compiled to *.lock.yml via gh aw compile

Use cases:
  • Automate issue triage and labeling
  • Generate daily status reports
  • Run scheduled code quality checks
  • Respond to slash commands in issues/PRs

Install CLI: gh extension install github/gh-aw

🏷️ Frontmatter Anatomy

Every primitive starts with YAML frontmatter. Click each tab to see the pattern:

instructions frontmatter --- applyTo: "**" # glob pattern for file targeting --- # Body: coding standards, conventions, stack rules… # Global file: .github/copilot-instructions.md # Scoped files: .github/instructions/*.instructions.md
agent frontmatter --- description: "Generate a working Flask prototype from a spec…" # triggers invocation tools: [read, edit, search, execute] # permitted capabilities argument-hint: "Point to the spec or requirements file" # user guidance --- # Body: full system prompt with persona, rules, procedures…
prompt frontmatter --- description: "Parse customer input into a structured spec…" # triggers suggestion argument-hint: "Paste the customer email or meeting notes" # user guidance agent: "agent" # optional: agent mode --- # Body: structured instructions and output template…
skill frontmatter --- name: ms-agent-framework # identifier for cross-refs description: "Generate decoupled AI agents using…" # triggers invocation argument-hint: "Describe the agent's purpose" # caller context --- # Body: deep procedural guide with code templates, checklists…
hook definition (JSON) # Hooks use JSON, not markdown frontmatter { "name": "secrets-scanner", "description": "Block commits containing secrets", "event": "on_save", // when to trigger "steps": [...] // actions to execute }
agentic workflow frontmatter --- name: Daily Issues Report # workflow display name description: "Generate a daily summary of open issues" # what it does triggers: [schedule, workflow_dispatch] # when to run --- # Body: natural language instructions for the coding agent # Compile to .lock.yml: gh aw compile
🚀 Quick Start — Build Your Own

Start with instructions

Create .github/copilot-instructions.md with your coding standards. Use applyTo globs for scoped rules in .github/instructions/.

Add a prompt

Create .github/prompts/my-task.prompt.md with YAML frontmatter (description, argument-hint) and the prompt body.

Add an agent

Create .github/agents/my-agent.agent.md with a persona, tool permissions, and step-by-step procedure in the body.

Add skills for domain knowledge

Create .github/skills/my-skill/SKILL.md with deep domain instructions. Reference it from your agent.

Add hooks and workflows as needed

Hooks (.github/hooks/) enforce guardrails. Agentic workflows (.github/workflows/*.md) automate CI/CD with gh aw compile.

Explore the community

Browse github/awesome-copilot for hundreds of community-contributed agents, skills, hooks, and workflows you can install.
